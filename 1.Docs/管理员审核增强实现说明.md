## 1. 背景与问题诊断

管理员审核页面（`#/admin/review`）存在两个运行时 Bug：

### Bug 1：审核卡片不显示图片

**现象**：审核页每条记录的图片区域为空白，点击无反应。

**根因**：后端 `list_contributions()` 只返回 `image_paths`（磁盘相对路径，如 `Data/contributed/extend/...`），前端模板遍历 `item.images`（不存在的字段）。此外路径拼接到 URL 时存在双重 `Data/` 前缀：存储路径以 `Data/` 开头，`/model-data` 挂载点又指向 `2.Model/Data/`，导致最终解析为 `2.Model/Data/Data/...` 的无效路径。

```
存储路径:  Data/contributed/extend/Tomato___Early_blight/ext_xxx.jpg
错误 URL:  /model-data/Data/contributed/... → 2.Model/Data/Data/... ❌
正确 URL:  /model-data/contributed/...       → 2.Model/Data/contributed/... ✅
```

### Bug 2：病害名称显示英文

**现象**：用户贡献页（`#/contribute`）记录表格中，扩展模式下的病害名显示为 `Early blight`（英文），应显示 `番茄早疫病`（中文）。

**根因**：后端不返回中文名称，前端只能对 `existing_class`（如 `Tomato___Early_blight`）做字符串拆分得到英文名。审核页虽有客户端 `cnName()` 函数从 `class_labels.json` 映射，但依赖浏览器端异步 fetch，在网络延迟或缓存问题下不稳定。

---

## 2. 修复方案总览

核心思路：**后端统一处理中文名映射 + 图片 URL 构造**，前端直接消费。

| 层 | 文件 | 改动 |
|----|------|------|
| 后端 | `collector.py` | `list_contributions()` 新增 `label_cn`/`crop_cn`/`images`/`time` 字段；修正 URL 路径；`get_stats()` 补 `approved`/`rejected` |
| 后端 | `collector.py` | 新增 `_load_cn_map()` 类方法，缓存 `class_labels.json` 映射 |
| 前端 | `contribute/page.js` | 记录表格优先使用 `label_cn` 显示中文名 |
| 前端 | `review/page.js` | `displayName()` 直接使用后端 `label_cn`；时间字段改为 `submit_time` |

## 3. 后端改动

### 3.1 collector.py — list_contributions()

**文件**：`3.Web/Backend/app/services/collector.py`

每条贡献记录在返回前附加以下字段：

| 新增字段 | 说明 | 示例 |
|---------|------|------|
| `label_cn` | 中文病害全名 | `番茄早疫病`（extend）/ `小麦 — 赤霉病`（new） |
| `crop_cn` | 中文作物名 | `番茄` |
| `disease_cn` | 中文病害简称 | `早疫病` |
| `images` | 完整 `/model-data/` URL 数组 | `["/model-data/contributed/extend/Tomato___Early_blight/ext_xxx_001.jpg"]` |
| `thumbnail` | 首张缩略图 URL | 同 `images[0]` |
| `time` | 提交时间别名 | `2026-07-23 21:02:04` |
| `created_at` | 提交时间别名 | 同上（兼容不同前端引用方式） |

#### 中文名映射逻辑

```python
cn_map = self._load_cn_map()  # { 'Tomato___Early_blight': {'cn':'番茄早疫病','crop':'番茄','disease':'早疫病'}, ... }

if mode == 'extend' and existing_class:
    label_cn = cn_map[existing_class]['cn']         # → "番茄早疫病"
    crop_cn  = cn_map[existing_class]['crop']       # → "番茄"
    disease_cn = cn_map[existing_class]['disease']  # → "早疫病"

elif mode == 'new':
    label_cn = f"{crop_name} — {disease_name}"     # → "小麦 — 赤霉病"
```

#### 图片 URL 构造

```python
for p in image_paths:           # p = "Data/contributed/extend/Tomato___Early_blight/ext_xxx.jpg"
    normalized = p.replace("\\", "/")
    if normalized.startswith("Data/"):
        normalized = normalized[len("Data/"):]       # → "contributed/extend/Tomato___Early_blight/ext_xxx.jpg"
    url_paths.append(f"/model-data/{normalized}")    # → "/model-data/contributed/extend/Tomato___Early_blight/ext_xxx.jpg"
```

### 3.2 collector.py — get_stats()

新增两个计数字段，供审核页统计卡片使用：

```python
"approved": sum(1 for r in records if r.status == APPROVED),    # 已采纳提交数
"rejected": sum(1 for r in records if r.status == REJECTED),    # 已驳回提交数
```

原有 `approved_images`（已采纳图片数）和 `pending_count` 保持不变。

### 3.3 collector.py — _load_cn_map()

新增类级别缓存方法，首次调用时从 `2.Model/Data/class_labels.json` 加载映射表，后续调用直接返回缓存：

```python
_cn_map_cache: Optional[dict] = None  # 类变量

@classmethod
def _load_cn_map(cls) -> dict:
    if cls._cn_map_cache is not None:
        return cls._cn_map_cache
    with LABELS_PATH.open("r", encoding="utf-8") as f:
        data = json.load(f)
    cls._cn_map_cache = {item["en"]: item for item in data}
    return cls._cn_map_cache
```

## 4. 前端改动

### 4.1 用户端 contribute/page.js

**文件**：`3.Web/Frontend/js/features/contribute/page.js`

记录表格的"病害"列从字符串拆分为使用后端返回的中文名：

```javascript
// 旧代码（仅拆分英文）
{{ r.existing_class?.split('___')?.[1]?.replace(/_/g, ' ') || r.existing_class }}

// 新代码（优先中文名，逐级回退）
{{ r.label_cn || r.disease_cn || r.existing_class?.split('___')?.[1]?.replace(/_/g, ' ') || r.existing_class }}
```

同时在中文名下方显示作物标签（`r.crop_cn`），方便按作物浏览。

### 4.2 管理员审核 review/page.js

**文件**：`3.Web/Frontend/js/features/admin/review/page.js`

#### 4.2.1 中文名显示

新增 `displayName()` 函数，直接使用后端返回的 `label_cn`：

```javascript
function displayName(item) {
  return item.label_cn || item.cn || cnNameFallback(item);
}
```

保留 `cnNameFallback()` 作为降级方案（向后兼容旧数据），通过 `CLASS_MAP`（从 `class_labels.json` 异步加载）兜底。

#### 4.2.2 时间字段

```javascript
// 旧代码（两个不存在的字段）
{{ item.time || item.created_at || '-' }}

// 新代码（优先 submit_time，回溯别名）
{{ item.submit_time || item.time || item.created_at || '-' }}
```

#### 4.2.3 图片显示（无需改动）

审核页模板中 `item.images` 的 `v-for` 和 Lightbox 点击逻辑无需修改——后端已补充该字段，前端自动生效。

```html
<!-- 此段代码本身正确，之前因后端不返回 images 而失效 -->
<img v-for="(img, i) in (item.images||[]).slice(0,6)" :key="i"
  :src="img" @click="openLightbox(item.images, i)" />
```

## 5. 数据流

```
用户提交 (POST /api/contribute)
    │
    ▼
collector_service.submit()
    ├─ 保存图片 → 2.Model/Data/contributed/{extend|new}/...
    └─ 写入索引 → contributions.json（不含中文名）

管理员/用户查看 (GET /api/contribute/list 或 /api/admin/contributions)
    │
    ▼
collector_service.list_contributions()
    ├─ 读取 contributions.json
    ├─ _load_cn_map() → 从 class_labels.json 获取 {en: {cn, crop, disease}}
    ├─ 为每条记录补充：
    │   ├─ label_cn / crop_cn / disease_cn（中文名）
    │   ├─ images / thumbnail（完整 /model-data/ URL 数组）
    │   └─ time / created_at（别名）
    └─ 返回给前端

前端渲染
    ├─ contribute/page.js → 使用 label_cn 显示中文名
    └─ review/page.js     → 使用 images 显示缩略图 + Lightbox
                            → 使用 label_cn 显示中文名
```
