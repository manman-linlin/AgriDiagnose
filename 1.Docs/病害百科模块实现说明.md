# 病害百科模块实现说明

> 更新日期：2026-07-23  
> 数据范围：模型全部 38 类（26 个病虫害状态 + 12 个健康状态）

## 1. 模块目标

病害百科为诊断结果提供可检索、可维护、可追溯的知识说明。用户可以按作物、类别和关键词浏览；管理员可维护词条、关联模型类别并更新配图。

## 2. 数据结构

- `id`：唯一词条标识
- `class_en`：可选的模型类别关联
- `name_cn` / `name_en`：中英文名称
- `crop_cn` / `category` / `risk_level`：作物、分类和风险
- `symptom_summary` / `symptoms`：症状摘要与典型症状
- `pathogen` / `epidemiology` / `differentiation`：病原、发病规律与鉴别要点
- `prevention` / `treatment`：预防和处置建议
- `sources` / `updated_at`：资料来源和更新日期

## 3. 已实现功能

- 用户端：38 类覆盖、分页、搜索、作物与类别筛选、详情展示
- 管理端：新增、编辑、删除、图片上传、模型类别关联、分页和批量导入
- 后端：严格校验、唯一性检查、原子写入、图片验证与生命周期清理

## 4. 图片策略

1. 管理员上传图片优先显示。
2. 未上传时使用 `5.Test/image/` 中的类别示例图。
3. 上传图片限制 5MB，并验证真实图像内容。

## 5. 内容质量原则

- 每个病害应尽量提供独立的病原、发病条件和鉴别要点。
- 药剂使用必须以当地登记标签和农技指导为准。
- 健康分类不等于排除未知病害或早期感染。

## 6. 验收清单

- [ ] 38 个模型类别均有唯一关联词条
- [ ] 新上传图片刷新后仍优先显示
- [ ] 非图片、超大图片和未知词条上传会被拒绝
- [ ] JSON 批量导入先预检再应用
