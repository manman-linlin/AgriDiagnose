## 1. 模块概览

数据收集模块允许用户上传农作物叶片图片并标注病害类型，实现：

- **扩展已有类别**：为现有 38 类病害补充更多真实场景样本
- **新增病害类型**：提交训练数据中不存在的新病害
- **管理员审核**：管理员登录后可审核（通过/驳回+备注）用户提交的数据
- **贡献统计**：实时展示社区贡献总量

```
用户浏览器                              后端服务
┌─────────────────┐    POST/GET     ┌──────────────────┐
│ page-contribute │ ←─────────────→ │ /api/contribute/* │
│ (Vue 组件)       │   JSON/FormData │ /api/admin/*      │
└─────────────────┘                 └────────┬─────────┘
                                             │
                                    ┌────────▼─────────┐
                                    │ CollectorService  │
                                    │ .submit()         │
                                    │ .list_contributions()
                                    │ .review()         │
                                    │ .get_stats()      │
                                    └────────┬─────────┘
                                             │
                              ┌──────────────▼──────────────┐
                              │ 2.Model/Data/contributed/   │
                              │ ├─ contributions.json        │
                              │ ├─ extend/   (扩展已有类别)   │
```

## 3. 后端实现

### 3.1 CollectorService 核心类

**文件**：`3.Web/Backend/app/services/collector.py`

模块级单例 `collector_service`，Web 后端通过 `from app.services.collector import collector_service` 直接引用。

#### 方法一览

| 方法 | 签名 | 功能 |
|------|------|------|
| `submit()` | `(mode, images, existing_class, crop_name, disease_name, ...) → dict` | 保存图片 + 写入索引 |
| `list_contributions()` | `(status=None) → list[dict]` | 按审核状态筛选，时间倒序 |
| `review()` | `(id, approved, notes) → bool` | 管理员审核一条记录 |
| `get_stats()` | `() → dict` | 总提交/总图片/已采纳/扩展/新增/待审核 |
| `get_classes()` | `() → list[dict]` | 读取 class_labels.json 返回 38 类 |

#### `submit()` 执行流程

```
1. 生成贡献 ID（UUID 前 12 位）
2. 根据 mode 确定目标目录
   - extend → contributed/extend/{英文类名}/
   - new    → contributed/new/{作物}___{病害}/
3. 逐张保存图片（RGB JPEG，quality=92）
   - extend: ext_{类名简写}_{时间戳}_{序号}.jpg
   - new:    new_{时间戳}_{序号}.jpg
4. new 模式下写入 metadata.json（作物名/病害名/描述/提交时间）
5. 构建 Contribution 记录
6. 写入 contributions.json（头部插入，保证时间倒序）
7. 返回记录 dict
```

#### 图片处理细节

- 统一转为 RGB 色彩空间（支持 RGBA/LA/P 模式的透明背景图）
- JPEG 格式保存，质量参数 92
- 文件路径存储为相对 `2.Model/` 的正斜杠路径，兼容 URL 访问

#### JSON 索引文件

- 读取时处理文件不存在 → 返回 `[]`
- 写入时 `ensure_ascii=False` + `indent=2` 保证中文可读
- 原子写入：先写 `.tmp` 再 `replace()`，防止写一半崩溃导致索引损坏

### 3.3 API 端点

全部新增接口汇总（`main.py`）：

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| `POST` | `/api/contribute` | 无 | 提交数据贡献 |
| `GET` | `/api/contribute/list` | 无 | 贡献记录列表，`?status=pending\|approved\|rejected` |
| `GET` | `/api/contribute/stats` | 无 | 贡献统计数据 |
| `GET` | `/api/contribute/classes` | 无 | 已有 38 类中英文列表 |
| `POST` | `/api/admin/login` | 无 | 管理员密码登录 |
| `POST` | `/api/admin/logout` | Bearer | 退出登录（销毁 token） |
| `POST` | `/api/admin/review/{id}` | Bearer | 审核一条贡献 |
| `GET` | `/api/admin/contributions` | Bearer | 管理员视角贡献列表 |

#### POST `/api/contribute` 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `mode` | string | 是 | `"extend"` 或 `"new"` |
| `files` | File[] | 是 | 图片文件列表 |
| `existing_class` | string | extend 必填 | 英文类名，如 `Tomato___Early_blight` |
| `crop_name` | string | new 必填 | 作物名称 |
| `disease_name` | string | new 必填 | 病害名称 |
| `disease_description` | string | 否 | 病害描述 |
| `location` | string | 否 | 拍摄地点 |
| `photo_date` | string | 否 | 拍摄时间（YYYY-MM-DD） |
| `notes` | string | 否 | 备注 |

#### POST `/api/admin/review/{id}` 请求体

```json
{
  "approved": true,
  "notes": "图片清晰，审核通过"
}
```

### 3.4 管理员认证

采用轻量级 Token 认证，不使用数据库：

```
登录流程：
  密码 → POST /api/admin/login → 验证 → 生成 token(secrets.token_hex(32))
  → 存入内存 ADMIN_TOKENS set → 返回 token

请求保护接口：
  请求头 Authorization: Bearer <token>
  → Depends(_require_admin) 提取验证
  → 有效: 正常处理
  → 无效: 401 "请先登录管理员账号"

退出：
  POST /api/admin/logout → 从 ADMIN_TOKENS 移除 token

前端持久化：
  token 存入 localStorage('agridiag_admin_token')
  页面刷新后自动恢复登录态
  401 响应时自动清除过期 token
```

- 默认密码：`admin123`（环境变量 `ADMIN_PASSWORD` 可覆盖）
- Token 仅在内存中，服务重启后全部失效
- 适配课程设计演示场景，无需额外数据库

## 4. 数据存储

### 4.1 目录结构

```
2.Model/Data/contributed/                  ← 用户贡献根目录（Git 追踪，但 .gitkeep 占位）
│
├─ contributions.json                      ← 贡献索引（所有记录的时间倒序列表）
│
├─ extend/                                  ← 扩展已有类别
│   ├─ Tomato___Early_blight/
│   │   ├─ ext_Toma_Early_blight_20260721_140530_001.jpg
│   │   └─ ext_Toma_Early_blight_20260721_140530_002.jpg
│   └─ Corn___Common_rust_/
│       └─ ext_Corn_Common_rust_20260721_150000_001.jpg
│
└─ new/                                     ← 新增病害类型
    ├─ 水稻___稻瘟病/
    │   ├─ metadata.json                    ← {"crop_name":"水稻","disease_name":"稻瘟病",...}
    │   └─ new_20260721_145000_001.jpg
    └─ 小麦___赤霉病/
        ├─ metadata.json
        └─ new_20260721_151500_001.jpg
```

### 4.2 图片命名规则

| 模式 | 格式 | 示例 |
|------|------|------|
| extend | `ext_{类名简写}_{时间戳}_{序号}.jpg` | `ext_Toma_Early_blight_20260721_140530_001.jpg` |
| new | `new_{时间戳}_{序号}.jpg` | `new_20260721_145000_001.jpg` |

- **类名简写**：取英文类名前 4 字符 + `_` + 病害部分前 16 字符
- **时间戳**：`YYYYMMDD_HHMMSS` 格式
- **序号**：3 位补零（`001`, `002`, …）

### 4.3 索引文件格式

`contributions.json` 是一个 JSON 数组，每条记录包含：

```json
[
  {
    "id": "uuid前12位",
    "mode": "extend | new",
    "submit_time": "YYYY-MM-DD HH:MM:SS",
    "status": "pending | approved | rejected",
    "existing_class": "英文类名（extend模式）",
    "crop_name": "作物名（new模式）",
    "disease_name": "病害名（new模式）",
    "disease_description": "描述（new模式）",
    "image_count": 3,
    "image_paths": ["相对路径1", "相对路径2", ...],
    "location": "拍摄地点",
    "photo_date": "YYYY-MM-DD",
    "notes": "备注",
    "review_notes": "审核备注"
  }
]
```

- 最新记录在数组头部
- 相对路径以 `Data/contributed/` 开头，统一使用正斜杠
- 前端通过 `/model-data/{相对路径}` 访问图片

## 5. 前端实现

### 5.1 页面生命周期

```
页面加载 (created)
    │
    ├─ 检查 store.diagnosis.result
    │   └─ 有 → 预填 selectedClass（来自诊断页跳转）
    │
    ├─ loadClasses()    → GET /api/contribute/classes → classList
    └─ loadData()       → GET /api/contribute/list + /api/contribute/stats
                            │
                            ├─ records → 渲染记录表格
                            └─ stats   → 渲染统计卡片
```

### 5.2 组件状态机

```
[step=0] 模式选择
  ├─ 两张卡片并排（CSS Grid 1fr 1fr）
  ├─ 点击卡片 → 高亮选中（绿色边框 + ✓）
  └─ 点击"下一步" → [step=1]

[step=1] 表单
  ├─ 上传图片（多文件、缩略图预览、可删除、≤20张、≤10MB）
  ├─ 标注信息（扩展模式：搜索式下拉，新模式：文本输入）
  ├─ 补充信息（地点、时间、备注）
  ├─ 实时表单验证
  └─ 提交 → POST /api/contribute → Toast → 刷新 → 重置

[始终可见] 统计卡片 + 贡献记录表格
  ├─ 顶部 4 个统计数字卡片
  └─ 下方筛选 + 数据表格
```

#### 图片上传

- 支持 JPG/PNG 格式过滤
- 前端限制：最多 20 张、单张 ≤10MB
- 本地预览使用 `URL.createObjectURL()`，删除时 `revokeObjectURL()`
- 点击 `+` 触发表单中隐藏的 `<input type="file" multiple>`

#### 病害下拉选择器（扩展模式）

- 搜索框输入关键词 → 实时过滤 `classList`（匹配 cn / en / crop 字段）
- 下拉列表显示 "作物 — 中文病名" 格式
- 下方显示匹配数量

#### 表单验证

| 条件 | 验证规则 |
|------|---------|
| 未上传图片 | 提交按钮禁用 |
| extend 模式 + 未选类别 | 提交按钮禁用，下拉框红色边框 |
| new 模式 + 作物名为空 | 提交按钮禁用，输入框红色边框 + 错误提示 |
| new 模式 + 病害名为空 | 提交按钮禁用，输入框红色边框 + 错误提示 |

### 5.3 管理员审核 UI

```
顶部栏
  ├─ 未登录 → "🔒 管理员登录" 按钮 → 弹出模态框
  └─ 已登录 → "👤 管理员" 绿色徽章 → 点击退出

登录模态框
  ├─ 玻璃拟态卡片（居中、弹出动画）
  ├─ 密码输入框（type=password，Enter 提交）
  └─ 登录按钮 + 错误提示

记录表格（已登录时）
  ├─ ⏳ 审核中 → 显示 "✅ 采纳" "❌ 驳回" 按钮
  ├─ 点击审核 → 内联展开备注输入框
  │   ├─ textarea（审核备注，选填）
  │   └─ 确认 / 取消 按钮
  ├─ 提交审核 → POST /api/admin/review/{id} → Toast → 表格自动刷新
  └─ 非待审核状态 → 仅显示状态标签，无操作按钮
```

## 6. 数据流与跨页面协作

### 诊断页 → 数据贡献页

```
诊断页 (page-diagnose)
  │ POST /api/predict 成功
  │ store.diagnosis.result = { top1: { label_en, label_cn, ... }, top3, ... }
  │
  └─ 用户点击侧边栏"数据贡献"
        │
        ▼
数据贡献页 (page-contribute)
  │ created() 检测 store.diagnosis.status === 'done'
  │ → 自动设置 mode='extend', selectedClass = top1.label_en
  │ → 用户只需上传图片 → 提交
```

### 全局状态共享

通过 `window.AppStore`（Vue.reactive）跨越页面：

| 状态路径 | 用途 |
|---------|------|
| `store.diagnosis.result` | 诊断结果，贡献页预填使用 |
| `store.admin.loggedIn` | 管理员登录态，多个组件共享 |
| `store.admin.token` | Bearer token，api.js 自动附加 |
| `store.admin.showLoginModal` | 控制登录弹窗显隐 |

## 7. 与设计文档的差异

| 项 | 文档要求 | 实际实现 | 原因 |
|----|---------|---------|------|
| 拖拽上传 | 要求支持拖拽 | 仅点击上传 | 实现复杂度与课程周期权衡 |
| 上传进度条 | `XMLHttpRequest.upload.onprogress` | 未实现 | 同上 |
| 病害下拉框按作物分组 | `<optgroup>` 分组显示 | 平铺 + 搜索过滤 | 搜索过滤交互更灵活 |
| 记录行展开缩略图 | 点击展开查看已提交图片 | 未实现 | 贡献列表中显示缩略图 URL 已就绪 |
| JSON → SQLite | 文档建议迁移 | 继续使用 JSON 文件 | 文档自身也说"短期内可继续使用 JSON" |
| 管理员登录 | 文档未详细设计 | 密码 + Bearer Token 方案 | 自行补充，适配课程演示场景 |
| `schemas.py` | 文档要求新增 | Pydantic 模型直接写在 `main.py` | 模型数量少，单独文件意义不大 |
