## 1. 总体概述

### 1.1 当前系统状态

系统已基本成型，包含以下模块：

| 模块 | 状态 | 说明 |
|------|------|------|
| 智能诊断 | ✅ 已完成 | 图片上传 → ConvNeXt-Tiny 分类 → Top-3 结果展示 |
| AI 对话 | ✅ 已完成 | 硅基流动多模态 + SSE 流式 + 纠错机制 |
| 数据贡献 | ✅ 已完成 | 用户上传图片 + 标签，管理员审核 |
| 病害百科 | ✅ 已完成 | 38 类病害图鉴，搜索/筛选/详情抽屉 |
| 历史·统计 | ✅ 已完成 | 诊断记录 + ECharts 饼图统计 |
| 管理员登录 | ✅ 基础完成 | 密码登录 + Bearer Token，审核功能内嵌在贡献页 |

### 1.2 本次完善目标

| 完善点 | 核心内容 | 优先级 |
|--------|----------|--------|
| 管理员页面 + 用户系统 | 独立管理员后台 + 用户注册/登录 | 🔴 高 |
| 审核增强 | 图片预览、中文名称显示 | 🔴 高 |
| API-Key 配置 | 管理员可配置多模型 API-Key | 🔴 高 |
| 监控仪表盘 | 病虫害数据总览、图表展示 | 🟡 中 |
| 模型训练管理 | 一键重训练、类别监控、训练状态通知 | 🟡 中 |
| 百科管理 | 百科内容增删改查 | 🟡 中 |
| 用户页面 | 用户注册登录、个人中心 | 🟢 低 |

### 1.3 数据存储原则

**不使用数据库**，所有数据以 JSON 文件形式存储，统一放在 `3.Web/Backend/app/data/` 目录下：

| 文件名 | 用途 | 现有/新增 |
|--------|------|-----------|
| `encyclopedia.json` | 病害百科数据 | 现有 |
| `users.json` | 用户账号信息 | **新增** |
| `config.json` | 系统配置（API-Key 等） | **新增** |
| `training.json` | 训练任务状态 | **新增** |
| `2.Model/Data/contributions.json` | 贡献审核数据 | 现有 |

### 2.3 用户系统

#### 2.3.1 概述

- 普通用户可**注册账号**和**登录**
- 登录后可使用全部功能（诊断、对话、贡献、百科等）
- 未登录用户可浏览百科，但不能使用诊断和对话功能
- 用户数据存储在 `users.json` 中

#### 2.3.2 用户数据模型 (`users.json`)

```json
[
  {
    "id": "user_a1b2c3d4",
    "username": "zhangsan",
    "password_hash": "sha256_hash_here",
    "display_name": "张三",
    "email": "zhangsan@example.com",
    "role": "user",
    "created_at": "2026-07-22 10:30:00",
    "last_login": "2026-07-22 14:20:00",
    "stats": {
      "diagnosis_count": 15,
      "contribution_count": 3,
      "approved_count": 2
    }
  }
]
```

#### 2.3.3 用户角色

| 角色 | 权限 |
|------|------|
| `user` | 使用诊断、对话、贡献、百科功能 |
| `admin` | 用户权限 + 进入管理后台 + 审核 + 系统配置 |

#### 2.3.4 用户 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/user/register` | 用户注册 |
| `POST` | `/api/user/login` | 用户登录，返回 Token |
| `POST` | `/api/user/logout` | 退出登录 |
| `GET` | `/api/user/profile` | 获取当前用户信息（需登录） |
| `PUT` | `/api/user/profile` | 修改个人信息 |
| `GET` | `/api/admin/users` | 管理员获取用户列表 |
| `PUT` | `/api/admin/users/{id}` | 管理员修改用户权限 |

#### 2.3.5 前端实现

```
3.Web/Frontend/js/
├── features/
│   └── user/
│       ├── LoginPage.js       # 登录页面
│       ├── RegisterPage.js    # 注册页面
│       └── ProfilePage.js     # 个人中心页面
├── stores/
│   └── user.js                # 用户状态管理（替代/扩展当前 admin.js）
```

#### 2.3.6 密码安全

- 前端传输前使用 SHA-256 哈希
- 后端存储二次哈希（SHA-256 + 随机盐）
- 盐值存储在 `users.json` 同一条目中（`salt` 字段）

### 3.3 关键功能

#### 3.3.1 图片预览

- 审核卡片中直接显示缩略图（200×200）
- **点击缩略图弹出大图预览弹窗**（Lightbox），支持左右切换
- 图片通过 `/model-data/` 静态路径加载

#### 3.3.2 中文名称显示

- 扩展模式下：通过 `existing_class` 字段查询 `class_labels.json`，显示中文名
- 新增模式下：直接显示 `crop_name` + `disease_name`
- 格式：`中文名（英文类名）`，如：`番茄早疫病（Tomato___Early_blight）`

#### 3.3.3 批量审核

- 支持多选（勾选多条记录）
- 批量采纳 / 批量驳回
- 批量操作时统一填写审核备注

#### 3.3.4 审核详情

- 点击记录展开详细信息：
  - 提交者信息（用户名、提交时间）
  - 补充信息（拍摄地点、拍摄日期、备注）
  - 图片 EXIF 信息（如有）

### 3.4 后端增强

- 贡献记录返回时自动附加中文名映射
- 新增 `GET /api/admin/contributions/{id}` 获取单条记录详情（含图片列表）
- 新增 `POST /api/admin/review/batch` 批量审核接口

### 6.3 训练流程

```
管理员点击"开始训练"
    ↓
后端启动训练子进程 (train.py)
    ↓
写入 training.json → 状态: "training"
    ↓
前端轮询 GET /api/admin/training/status
    ├─ 用户页面检测到 training 状态 → 显示 "模型更新中" 横幅
    └─ 管理员页面显示实时进度
    ↓
训练完成 → 更新 meta.json → 状态: "idle"
    ↓
备份旧模型 → 替换权重文件 → 重载分类器
    ↓
前端收到完成通知 → 横幅消失
```
